03 | Featured case study

Modular steering wheel
input and display system.

ECD619 capstone project for the 2025-2026 Binghamton Motorsports Formula SAE Electric vehicle: a race-ready driver interface that moves telemetry, inputs, and diagnostics onto the steering wheel.

Problem

The previous Raspberry Pi dashboard was flexible, but it was not the right fit for a racecar driver interface: long 10-20 second boot times, higher power draw, inconsistent latency, frame-rate instability, and sensitivity to vibration and thermal cycling.

System architecture

The replacement system uses a real-time microcontroller architecture inside the steering wheel to display CAN bus data, capture driver inputs, and support faster startup. The steering wheel module is paired with right and left dashboard subsystems for E-stop, Ready-to-Drive, fault indication, voltmeter access, banana jacks, and RTD buzzer support.

6pushbuttons
2encoders
CANvehicle bus

Steering wheel module

Adafruit Feather M4 CAN Express, ST7796S SPI TFT display, matrix-scanned buttons, two rotary encoders, buck-regulated power, SD-card diagnostics, and CAN high/low through the quick-release interface.

Vehicle safety interface

Right and left dashboard modules keep FSAE EV safety functions visible and separated, including emergency stop, Ready-to-Drive signaling, fault LEDs, voltmeter, banana jacks, and buzzer circuitry.

Race operations goal

The long-term stretch goal is CAN interoperability with the team's Data Acquisition Module so steering-wheel display events and DAQ telemetry can be logged together for diagnostics and test-day analysis.
